# 专有概念

## IPO 认购类型

### 现金认购（Cash Subscription）
投资者使用自有现金参与 IPO 申购，不涉及融资。只需验证账户余额是否充足。

### 融资认购（Margin Subscription）
投资者通过融资（借贷）方式参与 IPO 申购，需要支付融资成本和利息。系统需额外处理：
- 融资金额计算
- 融资利率/成本设置
- 资金冻结与解冻
- 融资凭证生成

## 工作流系统

### ProcessDefinition - 流程定义
定义审核流程的节点结构，包括：
- `nodeId`: 节点 ID
- `prevId`: 上一节点
- `nextId`: 下一节点
- `processName`: 流程名称（与申请类型关联）

### ProcessTask - 流程任务
当前待处理的任务：
- `applicationId`: 关联的申请 ID
- `status`: 任务状态（TODO/DONE）
- `roleName`: 执行角色（如 ROLE_IC）

### ProcessRecord - 流程记录
已完成的审核记录：
- `checkNodeId`: 审核节点 ID
- `checker`: 审核人
- `optType`: 操作类型（EXECUTION/REJECT/CANCEL）
- `remark`: 审核意见
- `checkTime`: 审核时间

### 审核节点类型
- **START**: 起始节点
- **IC** (Introductory Check): 初步检查节点
- 其他节点由 ProcessDefinition 动态定义

## 凭证（Voucher）系统

### 双系统架构
系统同时维护两套凭证状态：
- **Neohas**: 资金管理系统，处理现金存取
- **Ayers**: 经纪系统，处理证券交易相关凭证

### 凭证生命周期
```
PENDING → SUCCEED (成功)
     ↓
   FAILED (失败，可重试)
```

### 凭证方向
- **D** (Deposit): 存入
- **W** (Withdraw): 取出

作废时方向反转（W↔D），生成反向凭证。

## 外部系统集成

### FINI 系统
中央结算系统的 IPO 认购接口：
- `addSubscriptionEntries`: 添加认购条目
- `invalidateSubscription`: 撤销认购
- 返回 `recordID` 用于追踪

### SCB 通道
SCB 银行的直接通道，使用 `stockCode#accountCode` 作为 recordId，不经过 FINI。

### Neohas 资金系统
处理现金的存入/取出通知：
- 贷款金额存入（融资认购）
- 认购金额扣款
- 退款处理

### Asset Management API
资产管理系统的冻结/解冻接口：
- `callFrozenWithdrawApi`: 执行冻结或取款操作
- 返回状态码判断成功/失败

## 消息事件

### RocketMQ 事件

| 事件类型 | 类 | 用途 |
|----------|-----|------|
| REFERENCE_UPDATE | FiniReferenceEvent | 参考数据更新通知 |
| APPROVE | FiniStatusEvent | 审核状态变更通知 |
| VOUCHER | FiniStatusEvent | 凭证状态变更通知 |
| ALLOTMENT | FiniAllotmentEvent | 配售结果通知 |
| STATUS | FiniStatusEvent | 申请状态变更通知 |

### Spring 内部事件
`FiniInnerEvent`: 系统内部事件，用于触发凭证状态检查等后续操作。

## 申请映射（Application Mapping）

由于外部系统（如 FINI）使用自己的 referenceId，系统通过 `TIpoApplicationMapping` 表维护映射关系：
- `sourceId`: 外部系统的 referenceId
- `source`: 来源系统标识
- `targetId`: 内部 applicationId
